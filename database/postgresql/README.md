# WorshipFlow - PostgreSQL/Supabase

Arquivos gerados a partir dos SQLs MySQL/MariaDB em `database/`.

Ordem de execucao no Supabase SQL Editor:

1. `001_schema.sql`
2. `002_seed_current_data.sql`
3. `003_seed_100_musicas_usuarios.sql`

Observacoes tecnicas:

- As colunas `enum(...)` do MySQL foram convertidas para `VARCHAR(30)` com `CHECK`.
- `AUTO_INCREMENT` foi convertido para `GENERATED BY DEFAULT AS IDENTITY`.
- `tinyint(1)` foi convertido para `BOOLEAN`.
- `longtext` foi convertido para `TEXT`.
- `INSERT IGNORE` foi convertido para `ON CONFLICT (...) DO NOTHING`.
- Os IDs explicitos dos seeds sao preservados; por isso os scripts atualizam as sequences com `setval`.
- Execute em um banco limpo para evitar divergencias de schema. Em producao, prefira transformar estes arquivos em migrations Flyway.
