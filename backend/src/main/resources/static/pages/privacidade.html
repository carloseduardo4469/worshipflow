Privacidade e proteção de dados

Política de Privacidade

Esta Política explica como o WorshipFlow coleta, utiliza, armazena e protege dados pessoais.

O documento segue princípios da LGPD e considera a proteção integral de crianças e adolescentes prevista no ECA e no ECA Digital.

1. Introdução

O WorshipFlow respeita a privacidade dos usuários. Os dados pessoais tratados pelo sistema são utilizados exclusivamente para viabilizar seu funcionamento.

Isso inclui cadastro, autenticação, organização de membros, escalas, músicas, ensaios e comunicação interna.

Esta Política usa linguagem simples para que usuários, administradores, responsáveis e avaliadores do TCC compreendam como os dados são tratados.

2. Quais dados são coletados

O WorshipFlow coleta apenas dados necessários ao funcionamento do sistema:

  • Nome completo.
  • E-mail.
  • Senha armazenada de forma protegida.
  • Telefone.
  • Informações ministeriais, como habilidades e função, quando informadas.
  • Foto de perfil, quando enviada pelo próprio usuário.

O sistema não coleta documentos pessoais, dados bancários, geolocalização, reconhecimento facial ou dados pessoais sensíveis.

3. Finalidade da coleta

Os dados são utilizados exclusivamente para:

  • Criar e autenticar contas de usuário.
  • Identificar membros, líderes e administradores dentro do WorshipFlow.
  • Organizar escalas, ensaios, músicas e comunicações internas.
  • Permitir recuperação de senha e suporte operacional.
  • Manter a segurança, integridade e rastreabilidade básica do sistema.

4. Base legal conforme a LGPD

O tratamento de dados pessoais no WorshipFlow se apoia principalmente em:

  • Execução de contrato ou procedimentos relacionados ao uso do sistema.
  • Legítimo interesse para operação segura e organizada da plataforma.
  • Cumprimento de obrigações legais, quando aplicável.

Quando o uso envolver menor de idade, o tratamento deve observar o melhor interesse da criança ou adolescente, com transparência, necessidade e participação de pais ou responsáveis quando cabível.

5. Como os dados são armazenados

Os dados são armazenados em banco de dados utilizado pela aplicação. O acesso é restrito a usuários autorizados conforme perfil de permissão.

As senhas não são armazenadas em texto legível. Elas são protegidas por mecanismo criptográfico de hash, de modo que administradores não conseguem visualizar a senha original cadastrada pelo usuário.

6. Como os dados são protegidos

O WorshipFlow adota medidas de segurança proporcionais ao seu contexto acadêmico e operacional, incluindo:

  • Autenticação por e-mail e senha.
  • Controle de acesso por perfil de usuário.
  • Restrição de funcionalidades administrativas no backend.
  • Armazenamento protegido de senhas.
  • Uso de dados apenas para a finalidade do sistema.

7. Direitos do titular dos dados

Nos termos da LGPD, o titular pode solicitar, quando aplicável:

  • Confirmação sobre a existência de tratamento de seus dados.
  • Acesso aos dados pessoais mantidos pelo sistema.
  • Correção de dados incompletos, inexatos ou desatualizados.
  • Eliminação, bloqueio ou anonimização de dados desnecessários ou tratados em desconformidade com a lei.
  • Informações sobre eventual compartilhamento de dados.
  • Revogação de consentimento, quando essa for a base legal utilizada.

8. Solicitação de correção ou exclusão

O usuário pode solicitar atualização, correção ou remoção de dados pessoais pelos canais de contato indicados nesta Política.

A exclusão poderá ser limitada quando houver necessidade de manter dados para segurança, cumprimento legal, auditoria, defesa de direitos ou continuidade operacional autorizada.

9. Uso de cookies e armazenamento local

O WorshipFlow pode utilizar recursos técnicos do navegador, como armazenamento local, para manter a sessão do usuário, preferências de tema e dados necessários ao funcionamento da interface.

Esses recursos não são usados para publicidade comportamental ou venda de dados.

10. Compartilhamento de dados com terceiros

O WorshipFlow não vende dados pessoais e não realiza compartilhamento comercial.

Os dados podem ser acessados apenas por usuários autorizados, administradores do sistema e provedores técnicos estritamente necessários para hospedagem, e-mail, banco de dados ou manutenção, sempre conforme a finalidade do sistema.

11. Tratamento de dados de menores de idade

Caso crianças ou adolescentes utilizem o WorshipFlow em contexto de ministério, o tratamento de dados deve respeitar a proteção integral, o melhor interesse do menor, a minimização de dados e a supervisão de pais, responsáveis ou lideranças autorizadas quando cabível.

O sistema não deve ser usado para expor menores a conteúdo inadequado, publicidade comportamental, exploração comercial, assédio, discriminação, violência ou qualquer prática contrária ao ECA, ao ECA Digital e à LGPD.
